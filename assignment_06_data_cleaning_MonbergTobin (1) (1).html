Why This Matters¶

This assignment gives you experience cleaning raw data and justifying your choices so others can understand your process.

Dataset Description¶

The dataset you'll be using is a detailed Airbnb listing file, available from Inside Airbnb.

Each row represents one property listing. The columns include:

  • Host attributes (e.g., host ID, host name, host response time)
  • Listing details (e.g., price, room type, minimum nights, availability)
  • Location data (e.g., neighborhood, latitude/longitude)
  • Property characteristics (e.g., number of bedrooms, amenities, accommodates)
  • Calendar/booking variables (e.g., last review date, number of reviews)

📌 The schema is consistent across cities, so you can expect similar columns regardless of the location you choose.

print(f"Missing bedrooms before: {df['bedrooms'].isnull().sum()}")

df['bedrooms'] = df.groupby('room_type')['bedrooms'].transform(lambda x: x.fillna(x.median()))
df['bedrooms'] = df['bedrooms'].fillna(df['bedrooms'].median())

✍️ Your Response: 🔧¶

  1. bedrooms and reviews_per_month

  2. I filled missing bedrooms with the median for each room type, since whole homes usually have more bedrooms than private rooms. Missing reviews_per_month were filled with 0 to show no review activity.

  3. Filling bedrooms with the median hides unusual cases, like very small or large properties, which can distort pricing and capacity estimates. Filling reviews with zero mixes new, inactive, or missing listings, making some appear worse than they are. Both methods make the data simpler but can misrepresent reality.
